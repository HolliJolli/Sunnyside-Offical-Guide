Guide Content

This section provides our development team with a comprehensive guide for securing Windows 10 systems. It includes a detailed checklist and step-by-step instructions vital for maintaining a secure development environment.

By following this structured checklist, we can ensure that our Windows 10 systems are thoroughly secured. It is essential for our team to adhere to these guidelines to achieve optimal security and maintain a reliable development environment.

Windows 10 Security Checklist

First Steps

  • Discuss README together and identify critical tasks.
  • Install Service Packs if available, using WSUS Offline Updater.
  • Download necessary programs: MalwareBytes, BitDefender, CCleaner, IOBitUninstaller, MSBA, ProcessExplorer, Wireshark.

Password Policy

  • Enforce password history: 24
  • Maximum password age: 60 days
  • Minimum password age: 1 day
  • Minimum password length: 10 characters
  • Password must meet complexity requirements: Enabled
  • Store password using reversible encryption: Disabled

Lockout Policy

  • Account lockout duration: 30 minutes
  • Account lockout threshold: 10 attempts
  • Reset account lockout counter after: 30 minutes

User and Group Settings

  • Set “User must change password at next logon” for all users.
  • Disable all unauthorized user accounts.
  • Restrict 'Administrators' and 'Guests' groups to authorized users only.

Network and Internet Settings

  • Disable Wi-Fi Sense features to prevent automatic connections to open hotspots.
  • Turn UAC to max and disable IPv6 in adapter settings.

Windows Services

  • Disable services like UPnP Device Host, Telnet, and Remote Registry.
  • Ensure critical services like Windows Event Collector are running.

Firewall Configuration

  • Disable inbound access for Microsoft Edge, MSN apps, and Xbox.

Security Policies

  • Disable SMB v1 under Windows Features.
  • Only share ADMIN$, C$, and IPC$ unless otherwise specified.

System Configuration

  • Turn off AutoPlay and disable OneDrive on Startup.
  • Enable screensaver timeout and configure screensaver security.

Audit and Logging

  • Enable auditing for success/failure for critical logon events.

Local Security Policies

  • Apply settings to enforce user security such as disabling guest accounts and enforcing password complexity.
  • Restrict NTLM traffic and configure network access security as specified.

For more details on each step, refer to the README or consult your cybersecurity team.
